## Что делает NOUZ

<div class="lead-panel">
  <p>NOUZ превращает Obsidian-базу в локальную карту знаний. Вместо папок с файлами агент получает заметки, связи, уровни и соседний контекст. Структура остаётся под вашим контролем, а NOUZ помогает поддерживать порядок в базе и готовить точный контекст для ИИ-агента.</p>
</div>

## Зачем это нужно

Когда база растёт, агенту всё труднее выбрать нужный контекст: он читает лишние файлы, пропускает важные связи, путает черновики с выводами и каждый раз собирает материал заново. NOUZ даёт ему аккуратные действия над базой: прочитать заметку, увидеть её место в графе, пройти по связям, предложить разметку и найти похожие материалы.

## Три режима

Начните с простого: Obsidian, YAML и граф. Семантику можно подключить позже, когда база вырастет и простых ссылок станет мало.

## Для команд

Проще всего попробовать NOUZ на Obsidian. Если база уже живёт в Notion, Confluence, Google Docs или GitHub, её не обязательно переносить: тот же подход можно подключить через MCP-коннекторы или отдельные адаптеры.

## Техническая основа

Граф строится сверху вниз. Каждый уровень отвечает за свою роль, а для больших баз можно добавить `meta_root` — якорный узел уровня 0, который собирает домены в одну систему и не участвует в семантических расчётах.

| Уровень | Тип | Что сюда помещать | Откуда знак |
| ------- | --- | ----------------- | ----------- |
| L1 | Ядро | Домен знаний — широкая область | Вручную или по эталонам |
| L2 | Паттерн | Тематическая рамка для дочерних модулей и квантов | Вручную или по эталонам |
| L3 | Модуль | Функциональная группа, проект | Вручную или по эталонам |
| L4 | Квант | Одна идея — гайд, исследование, пост | Тип артефактов детей + домен по контенту |
| L5 | Артефакт | Сырьё: логи, чаты, конфиги | Тип по эвристике содержимого |

## Быстро попробовать

```bash
pip install nouz-mcp
```

Проще всего начать с Obsidian. Укажите путь к vault и подключите NOUZ к MCP-клиенту. Режим **LUCA** работает без эмбеддингов; для PRIZMA и SLOI добавьте `config.yaml` и адрес сервиса эмбеддингов.

Пример режима с семантическим слоем:

```yaml
mode: prizma
```

Переменные окружения:

```bash
export OBSIDIAN_ROOT=/path/to/vault
export EMBED_API_URL=http://127.0.0.1:1234/v1
```

Запустите через MCP-клиент:

```json
{
  "mcpServers": {
    "nouz": {
      "command": "python",
      "args": ["-m", "nouz_mcp"],
      "env": {
        "OBSIDIAN_ROOT": "/path/to/vault"
      }
    }
  }
}
```
