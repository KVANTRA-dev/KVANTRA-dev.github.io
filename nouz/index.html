Three implementations Три реализации

One codebase, three levels of structure. Одна кодовая база, три уровня структуры.

Δ

LUCA

"The ancestor. Graph model. Clean." "Предок. Граф-модель. Чисто."
  • Pure DAG hierarchy, no embeddings Чистая DAG-иерархия, без эмбеддингов
  • YAML frontmatter read/write Чтение/запись YAML frontmatter
  • Entity formula formatting Форматирование формулы сущности
  • Lightweight, zero dependencies Легковесный, без зависимостей
Ψ

PRIZMA

"The prism. Full semantic model. Drift detection." "Призма. Полная семантическая модель. Детекция дрифта."
  • Embedding classification via LM Studio Классификация через эмбеддинги LM Studio
  • Sign inheritance + core_mix propagation Наследование знака + распространение core_mix
  • Semantic bridge detection Детекция семантических мостов
  • Drift warnings, suggest_metadata Предупреждения о дрифте, suggest_metadata
Σ

SLOI

"The skeleton. Strict 5-level hierarchy." "Скелет. Строгая 5-уровневая иерархия."
  • Enforced L1→L2→L3→L4→L5 Строгий L1→L2→L3→L4→L5
  • Level violation detection Детекция нарушений уровней
  • Recursive formula updates Рекурсивное обновление формул
  • Obsidian callout block sync Синхронизация callout-блоков Obsidian

03 · quick start быстрый старт

Start in 3 minutes Запуск за 3 минуты

NOUZ runs locally via MCP stdio. Python 3.10+, an Obsidian vault, and optionally LM Studio for embeddings. Define your semantic cores in config.yaml — these are your axes of meaning. NOUZ работает локально через MCP stdio. Python 3.10+, Obsidian-хранилище, и опционально LM Studio для эмбеддингов. Определите ваши ядра в config.yaml — это ваши оси смысла.

Pick a mode: LUCA for pure graph, PRIZMA for full semantics, SLOI for strict hierarchy. All expose the same MCP tools. Выберите режим: LUCA для чистого графа, PRIZMA для полной семантики, SLOI для строгой иерархии. Все предоставляют одинаковые MCP-инструменты.

How it works in practice Как это работает на практике

Ψ

Classify new notes Классификация новых заметок

Write in Obsidian. Run suggest_metadata. NOUZ proposes sign and core_mix — you add to YAML frontmatter manually. Пишете в Obsidian. Запускаете suggest_metadata. NOUZ предлагает sign и core_mix — вы добавляете в YAML вручную.

prizma / sloi mode only · LM Studio required только prizma / sloi · требуется LM Studio
Δ

Audit your knowledge Аудит знаний

Run recalc_signs to update all notes in DB. Query SQLite to see distribution: % of Ψ vs Δ vs Σ. Запустите recalc_signs для обновления всех заметок в БД. Запросите SQLite для просмотра распределения: % Ψ vs Δ vs Σ.

works in all modes · shows knowledge drift работает во всех режимах · показывает дрейф знаний
Σ

Find semantic neighbors Поиск семантических соседей

Run suggest_parents to find related notes by meaning, not folder name. Запустите suggest_parents для поиска связанных заметок по смыслу, не по папке.

prizma / sloi only · LM Studio required только prizma / sloi · требуется LM Studio

⚠️ What NOUZ is NOT ⚠️ Чем NOUZ НЕ является

  • It does NOT auto-write to your notes — you run commands, NOUZ suggests, you decide what to add. NOUZ НЕ пишет в заметки автоматически — вы запускаете команды, NOUZ предлагает, вы решаете что добавить.
  • It does NOT require cloud API — runs 100% locally with LM Studio (free, open source). NOUZ НЕ требует облачного API — работает 100% локально с LM Studio (бесплатно, open source).
  • LUCA mode works WITHOUT embeddings — just YAML frontmatter and links. Режим LUCA работает БЕЗ эмбеддингов — только YAML frontmatter и ссылки.
